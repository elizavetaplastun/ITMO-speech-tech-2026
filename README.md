# ITMO-speech-tech-2026
Для выполнения работ вам понадобятся файлы блокнотов (labN.ipynb) и файлы из папки resources.
Файлы будут догружаться в репозиторий в течение курса. 

# Как работать с репозиторием:

## Вариант 1: Самый простой (Скачать архивом)
Этот способ не требует установки программ. Вы просто скачиваете текущую версию файлов.
- Откройте ссылку на репозиторий.
- Нажмите на зеленую кнопку «<> Code» в правой части экрана.
- Нажмите «Download ZIP».
- Распакуйте архив у себя на компьютере.

Минус: Чтобы получить обновления, придется каждый раз скачивать архив заново.

## Вариант 2: (Через Git)
Если хотите обновлять файлы одной командой, установите Git.
### 1. Установите git:

**Для Windows**:

Выполните в терминале: `winget install --id Git.Git -e --source winget`

>**Важно**: После установки перезапустите терминал, чтобы команда git начала распознаваться.

**Для Mac OS**:

Выполните в терминале:
`git --version`

> Если Git уже есть, покажется версия. Если его нет, macOS сама выкинет всплывающее окно с предложением установить "Инструменты командной строки Xcode". Нужно просто нажать «Установить» и дождаться завершения.

### 2. Склонируйте репозиторий:

- **Windows**: Нажмите правой кнопкой мыши в любой папке и выберите «Git Bash Here» (или найдите Git Bash в меню Пуск)
- **Mac**: Продолжаем работать в терминале

Перейдите в папку, где будет лежать проект. 

`cd "путь к нужной директории"`

Далее выполните следующую команду: 

`git clone https://github.com/elizavetaplastun/ITMO-speech-tech-2026.git`

После этого на компьютере появится папка проекта "ITMO-speech-tech-2026", а внутри — все нужные файлы.

Можно начинать работу!

### 3. Обновление проекта:
Если вам нужно догрузить новые файлы, то в терминале перейдите в папку проекта 

`cd "ваш путь/ITMO-speech-tech-2026"`

Далее выполните команду: 

`git pull`

Все новые файлы появятся автоматически.

### 4. Выполнение лабораторных работ и защита:

Для выполнения вы можете выбрать любой удобный вам инструмент. Для сдачи работ убедитесь, что можете отправить ссылку на ваш проект (google colab, git репозиторий)

На практических занятиях по расписанию будут происходить защиты лабораторных работ. Перед защитой убедитесь, что ваш проект запускается и проходит все тесты из файла.
